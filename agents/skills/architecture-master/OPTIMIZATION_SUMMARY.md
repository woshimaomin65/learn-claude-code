# Architecture Master Skill - Optimization Summary

## Skill 创建概述

本 skill 旨在创建一个"顶级架构师与多语言专家"角色，专门用于指挥大型项目的编程工作。

## 设计决策

### 1. 核心原则设计

选择了四个核心原则作为 skill 的指导方针：

1. **架构先行原则**: 强调在编码前进行充分的架构设计
2. **等效转换原则**: 确保语言迁移时的功能一致性
3. **增量构建原则**: 采用迭代方式降低风险
4. **工具增强原则**: 充分利用 MCP 工具提高效率

### 2. 工作流程设计

设计了五个阶段的标准工作流程：

1. **项目扫描与理解**: 使用 get_project_map 工具
2. **架构设计与规划**: 制定实施计划
3. **骨架搭建**: 使用 batch_file_operation 创建基础结构
4. **核心功能实现**: 使用增量编辑工具
5. **验证与优化**: 确保质量

### 3. 多语言能力

包含了三种主要语言转换的对照表：
- Python ↔ JavaScript/TypeScript
- Python ↔ Go
- Python ↔ Rust

这些对照表帮助在语言迁移时保持语义一致性。

### 4. 任务模板

提供了三种常见任务的标准化模板：
- 项目迁移（语言 A → 语言 B）
- 新功能开发
- 大规模重构

## 预期使用场景

1. **新项目启动**: 从零开始设计并构建大型系统
2. **语言迁移**: 将现有系统从一种语言迁移到另一种
3. **架构重构**: 重新设计现有系统的架构
4. **代码审查**: 分析项目结构并提出改进建议
5. **技术咨询**: 提供架构设计和技术选型建议

## 与 MCP 的协同

此 skill 设计为与 mcp-expert-programmer 协同工作：

| Skill 阶段 | 使用的 MCP 工具 |
|-----------|----------------|
| 项目扫描 | get_project_map |
| 骨架搭建 | batch_file_operation |
| 代码修改 | apply_incremental_edit |
| 批量操作 | batch_file_operation |

## 未来优化方向

1. **扩展语言支持**: 添加更多编程语言的特性对照
2. **架构模式库**: 内置常见架构模式的模板
3. **性能基准**: 添加性能测试和优化指南
4. **安全检查**: 集成安全编码实践检查

## 测试建议

1. 使用真实项目进行端到端测试
2. 验证语言迁移的功能等效性
3. 测试大规模重构的准确性
4. 收集用户反馈持续改进
